    Wstęp

    Witamy w Ebexo. Możesz tutaj znaleźć informacji o tym jak zintegrować się z systemem Ebexo API.

    Ebexo API jest oparte o architekturę REST i wykorzystuje standardowe zapytania HTTP w komunikacji. Zarówno żądania jak i odpowiedzi korzystają z formatu JSON. W zależności od konfiguracji sklepu, stosowanie protokołu HTTPS może być wymuszone. Każde zapytanie musi zawierać identyfikator sesji otrzymany podczas logowania do API. Zachęcamy do skorzystania z dema systemu Ebexo, które zwiera w pełni funkcjonalne API, co umożliwia błyskawiczne uruchomienie i przetestowanie działania systemu.

    Dostęp do API jest pod adresem http://ebexo.dev/api/json.

    <?php
$data = array("action"=>"login,
    "login" => "user_login",
    "pass" => "user_password");

$url_send = "http://ebexo.dev/api/json";

function sendCurl($data, $url) {
$data = json_encode($data);
    $ch = curl_init($url);
    curl_setopt($ch,CURLOPT_CUSTOMREQUEST, "POST");
    curl_setopt($ch,CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch,CURLOPT_POSTFIELDS,array("json"=>$data));
    curl_setopt($ch,CURLOPT_FOLLOWLOCATION,1);
    $result = curl_exec($ch);
    curl_close($ch);
    return $result;
    }

$session_id = sendCurl($data,$url_send);

    Zamówienia

    Zmówienia to zbiór produktów, ich odmian, które zakupuje klient. Zamówienie również określa stan w którym się znajduje - oczekujące na wpłatę, zwrócone, wysłane. Wszystkie zamówienia są przetrzymywane w bazie do czasu ich usunięcia, więc można bez problemu utworzyć kompletną listę zamówień danego klienta.

    Oprócz pobierania zamówień Ebexo API daje możliwość zmiany statusu, dodanie uwagi, faktury, usunięcie faktury z zamówienia, dodania produktu do zamówienia czy wreszcie utworzenie nowego zamówienia. Data jest podawana w formacie YYYY-MM-DD HH:MM:SS

    Obiekt ApiOrder posiada nastepujące pola:

    Klucz Typ Opis
    id Integer ID zamówienia
    client ApiOrderClient Obiekt ApiOrderClient - klient składający zamówienie
    client_delivery ApiClientDelivery Obiekt ApiClientDelivery - dane do dostawy
    date String Data złożenia zamówienia
    price Float Wartość zamówienia do zapłaty
    price_nopromo Float Cena zamówienia bez rabatu na całe zamówienie (np. kupon rabatowy)
    rebate Float Wartość rabatu - rabat końcowy na zamówienie, kupon rabatowy
    rebate_percent Float Wartość procentowa rabatu - rabat końcowy na zamówienie, kupon rabatowy
    rebate_items Float Suma rabatów udzielonych na produkty z zamówienia
    currency String Waluta rozliczeniowa
    exchange Float Kurs waluty względem waluty bazowej sklepu
    lang String Język zamawiającego
    weight Float Waga zamówienia
    is_fvat Choise Czy do zamówienia należy wystawić fakturę vat: 0 - nie, 1 - tak
    payment ApiPayment Obiekt ApiPayment - szczegoły płatności
    delivery ApiDelivery Obiekt ApiDelivery - szczegoły dostawy
    status ApiStatus Obiekt ApiStatus - szczegóły statusu zamówienia
    is_gift Choise Czy zamówienie należy zapakować na prezent: 0/null - nie, 1 - tak
    client_notice String Uwagi klienta
    is_allegro Choise Czy zamówienie pochodzi z serwisu allegro.pl: 0 - nie, 1 - tak
    allegro ApiOrderAllegro Obiekt ApiOrderAllegro - szczegóły zamówienia z allegro
    is_phone Choise Czy zamówienie jest telefoniczne: 0 - nie, 1 - tak
    offer_id Integer Id oferty względem której zamówienie zostało złożone
    is_merged Choise Czy zamówienie zostało połączone z innym: 0 - nie, 1 - tak
    merged_id Integer ID zamówienia, które powstało w wyniku połączenia
    is_merged_master Choise Czy zamówienie jest wynikiem połączenia kilku zamówień w jedno: 0 - nie, 1 - tak
    merged_from Array Tablica obiektów ApiOrderMerged zamówień, które połączono
    is_discount_code Choise Czy użyto kodu rabatowego: 0 - nie, 1 - tak
    discount_code ApiDiscountCode Szczegóły użytego kodu rabatowego

    Produkty

    Produkty są jednym z podstawowych elementów każdego systemu eCommerce. Reprezentują one przedmioty, które są oferowane w sprzedaży klientowi. Te produkty mogą być przeglądane, dodane do koszyka, czy zamówione przez klieta (jeżeli są aktywne i ich stan magazynowy na to pozwala).

    price_netto_nopromo Double Ceba netto bez promocji

    is_mag Choise Czy produkt posiada cechy magazynowe: 0 - nie, 1 - tak
    is_active Choise Czy produkt ma być widoczny w sklepie: 0 - nie, 1 - tak
    is_disabled Choise Czy produkt jest chwilowo niedostępny (brak w magazynie): 0 - nie, 1 - tak

    new_howlong Date Data końca oznaczenia produktu jako nowość
    is_best Choise Czy produkt ma być oznaczony jako bestseller: 0 - nie, 1 - tak
